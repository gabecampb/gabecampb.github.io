This is a bytecode set designed for efficient interpretation in virtual machines, consisting of 8-bit instructions.

Current version: 0.22

Features

* Keyboard input
* Mouse input
* Touchscreen input
* Graphics API (compatible with modern and previous generation graphics APIs)
* A shading language, represented in bytecode
* GPU computing
* Multi-threading and a high level of control over threads
* File I/O, including recognition and selection of connected drives
* SIMD
* Multi-display support
* Controller support
* Device orientation and position support
* Image capture from display or cameras
* 2D/3D audio
* Sound transmission, absorption, and scattering
* Audio file creation and editing
* Audio recording
* Audio playback
* Video file creation and editing
* Image file creation and editing
* Video playback
* Image loading
* Ray tracing
* Networking

Future of the Project

I will continue to clean up the specification as much as possible, involving elaboration, reorganization, and making it more concise. Many minor changes to the specification will be made, but the bytecode set is almost complete and will change as little as possible in future versions. I have been working on a virtual machine since the release of version 0.12 which is mostly complete, save for a bit of updating to recent versions which I'm still working on. Forward compatibility for MABS programs will be ensured from version 1.0 onward, which I hope to release sometime soon.

Upcoming Changes

In version 0.23, I will be polishing things and adding a few things that will improve on the bytecode set's graphics capabilities. Planned features for version 0.23 includes the ability to use instance ID as array indices in vertex shaders, and the addition of push constants (root constants in DX12 terminology; these can offer a more efficient alternative to uniform buffers when updating a small amount uniform data frequently/between draw calls). I also intend to better define some limits related to the amounts of data that can be input to shaders (such as the maximum amount of uniform data can be used), I will make file streams become private to the threads that create them, and there are more improvements that will be made. I'm also considering adding support for indirect rendering, AABB geometry for ray tracing, and intersection shaders.
